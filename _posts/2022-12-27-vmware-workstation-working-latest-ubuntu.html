--- creation_epoch: 1672190856 lastmod_epoch: 1672190856 categories: ['Computer Problems'] title: VMware Workstation Working on Latest Ubuntu --- Latest Ubuntu (or Xubuntu) distros have kernel versions that don't compile nicely for VMware Workstation Professional 16.x (or 17). Errors like "vmnet-only/vm_assert.h:43:10: fatal error: stdarg.h: No such file or directory" or similar.

I found the solution via multiple searches. Compiling vmmon module fails on linux kernel 5.16.0-arch1-1

https://github.com/mkubecek/vmware-host-modules/blob/master/INSTALL

Some pre-req commands: 

sudo apt-get install make gcc g++ autoconf build-essential linux-libc-dev git

In the event that community forum content disappears here are the quick start steps (assumes VMware bundle already installed): 

mkdir /opt/vmware

cd /opt/vmware

git clone https://github.com/mkubecek/vmware-host-modules.git

cd vmware-host-modules/

# Change-out with your version here
git checkout workstation-16.2.5

make VM_UNAME=$(uname -r)

make install


systemctl restart vmware
systemctl status vmware

I ignored the error when restarting vmware. It still just worked.
